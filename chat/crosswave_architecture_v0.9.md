## 1. 基本設計方針

### 1.1 DBアクセス

- 各ウィンドウ / パネルは **独立して** open-logbook API を直接叩く
- ウィンドウ間でのデータ共有オブジェクトは持たない
- DBの正となるのは open-logbook (SQLite)

### 1.2 ウィンドウ間通信

- ウィンドウ / パネル間の直接参照は持たない（疎結合）
- 通知は `NotificationCenter` を使用する
- 通知名は本仕様書で定義し、全パネルが共通で使う

### 1.3 ボードの多重起動

- ボードは複数同時に開くことができる（例: パイルアップ時に複数のQSOボードを同時使用）
- 各ボードインスタンスは独立して動作する

## 3. ログボード仕様

### 3.1 DBとの関連

- DBの内容を指定のフィルタで取得・表示する
- デフォルトはフィルタ無し
- 使用API: `GET /api/qso`

**フィルタ項目（予定）**:
- コールサイン（前方一致）
- 日時（from / to）
- QSLステータス

### 3.2 更新トリガー

- アプリ起動時
- `qso.updated` 通知を受信したとき → `GET /api/qso` で再取得

### 3.3 ボード操作

| 操作 | 動作 |
|------|------|
| ツールバー「NEW QSO」ボタン / ⌘N | QSOボードを**新規モード**で開く |
| リスト行ダブルクリック | QSOボードを**編集モード**で開く（対象DBエントリを渡す） |
| ⌘\` | ボードローテーション |

## 4. QSOボード仕様

### 4.1 モード

| モード | 起動方法 | 保存API |
|--------|----------|---------|
| 新規 | NEW QSOボタン / ⌘N | `POST /api/qso` |
| 編集 | リスト行ダブルクリック | `PUT /api/qso/{id}` |

### 4.2 新規モード初期値

- DATE: 起動時の現在日付（`YY/MM/DD`）
- TIME: 起動時の現在時刻（`HH:MMJ` / `HH:MMU`）
- その他: 空欄

### 4.3 入力バリデーション

- CALLSIGN が空の場合、SAVEボタンを無効化

### 4.4 コールサイン補完

- CALLSIGN 入力が2文字以上になったら候補をポップアップ表示
- 候補ソース: `GET /api/callsign_cache?q={prefix}`
- 候補選択時: NAME / QTH / CODE を自動入力

### 4.5 保存後の動作

1. `POST` / `PUT` 成功
2. `NotificationCenter.post("qso.updated")` を発火
3. ボードを閉じる

### 4.6 その他UI

- Escキー: ボードを閉じる（変更ありの場合は確認ダイアログ）
- 全角→半角自動変換（入力時）

## 5. 通知仕様

### 5.1 通知一覧

| 通知名 | 発火元 | 受信元 | 意味 |
|--------|--------|--------|------|
| `qso.updated` | QSOボード（保存成功時）| ログボード | QSOデータが追加/更新された |

### 5.2 設計思想

- 本来はDB（open-logbook）側からプッシュされるべきイベント
- 現状は「更新したことを知っているのはQSOパネル」という事実を利用し、アプリ内通知で代替する
- 将来 open-logbook 側に WebSocket 等を実装した場合は、通知元をDB側に切り替え可能

## 6. 新ボード開発ガイドライン

新しいボードを追加する場合は以下の規約に従うこと。

### 6.1 DBアクセス規約

- open-logbook API を直接叩く
- 他のボードのViewModelやデータを参照しない

### 6.2 通知規約

- QSOデータを書き込んだ場合: `qso.updated` を発火する
- QSOデータを表示する場合: `qso.updated` を受信して再取得する

### 6.3 UI規約

- FloatingPanel（NSPanel）として実装する
- 複数同時起動を前提とした設計にする（シングルトンにしない）

### 6.4 親子関係

- コントロールボードがルート（親なし）
- 全てのボードは起動元の「親ボード」を持つ
- 典型的な階層:

```
コントロールボード  ← ルート
  └─ ログボード
        └─ QSOボード
              └─ ログボード(2)  ← コールサインフィルタ済み
```

- 子ボードは確定時に親ボードに値を返す
- 親ボードは受け取った値をどう使うかを自身の実装で決める

### 6.5 ボードの種別

| 種別 | 説明 | 例 |
|------|------|-----|
| コントロールボード | アプリのルート。DBの管理操作を担当。値は返さない | CSVインポート/エクスポート、ログボード起動 |
| データボード | 確定時に親へ `QSORecord` を返す | ログボード、QSOボード |
| サービスボード | 確定時に親へ任意の値を返す（DBエントリとは無関係） | RIGリスト、アンテナリスト |

- コントロールボードはDB操作完了後に `qso.updated` を発火する
- サービスボードが何を返すかはそのボードの実装に委ねる
- 返す値がない場合は `null` を返す
